This is a follow-up to REPLIT_BACKEND_PROMPT.md (v3 firmware support — drops table event_id, drop_captures classifier columns, verdict polling, bin config toggles). That prompt scaffolded the API surface and DB schema. This prompt implements the classifier itself and the rollout plan that lets us go live at near-zero AI cost and scale up.

Architectural context (changed since the first prompt)
The Sensor MCU no longer has an SD card. Captures are POSTed directly to /api/bin-module/drop-capture over the sensor's WiFi. The sensor keeps a PSRAM-backed in-memory queue (~50–100 JPEGs) that drains as uploads succeed, so short WiFi blips don't lose data. The backend doesn't need to do anything different for this — /api/bin-module/drop-capture is already idempotent on event_id. Just confirm that endpoint accepts up to ~200 KB JPEGs without choking and that it returns quickly so the sensor can flush its queue.

Strategy: three-phase rollout
Phase 0 — "Pass-through" — start here, $0 AI cost
Classifier worker accepts captures, stores them, computes their SHA-256 + perceptual hash.
All verdicts default to accepted=true, reason="auto_accepted", verdict_review_needed=true.
Build a /admin/review page so staff can hand-label the queue.
Goal: collect ~500–2,000 labeled images from real bins. Validate the pipeline end-to-end without spending on AI.
Phase 1 — "Cheap vision API" — enable once Phase 0 is stable, ~$0.001 per image
Use Claude Haiku 4.5 vision (claude-haiku-4-5-20251001) with prompt caching.
Cache results by perceptual hash (pHash) — re-uses verdicts for re-scans within 24 h.
Hard budget cap per day (env var) — falls back to Phase 0 behavior if exceeded.
Phase 2 — "Self-hosted fine-tune" — only if volume justifies, ~$0.0001 per image
Train a MobileNet/EfficientNet-Lite on the labels accumulated in Phase 0+1.
Serve via ONNX runtime. Keep Haiku as the fallback for low-confidence cases.
Don't build Phase 2 yet. The prompt below covers Phases 0 and 1.

3. Classifier worker
server/classifier/classify.ts:

import Anthropic from '@anthropic-ai/sdk';
import { imageHash } from 'image-hash';

const anthropic = new Anthropic();

const SYSTEM = `You analyze a single photo of an object dropped into a vape
recycling bin in a corner store. Output ONLY a JSON object — no prose, no
markdown:

{ "label": "vape" | "thc_vape" | "not_a_vape" | "uncertain",
  "confidence": <number 0..1>,
  "rationale": "<one short sentence>" }

Definitions:
- "vape": any disposable or rechargeable nicotine vape (Elf Bar, Vuse, Juul, etc.)
- "thc_vape": cannabis/THC cartridge, pen, or rechargeable battery + cartridge.
  These are typically darker, often have a glass cartridge with visible oil.
- "not_a_vape": anything else (lighter, gum, trash, finger, empty frame).
- "uncertain": visibility poor, partially occluded, motion blur, or genuinely ambiguous.

Err toward "uncertain" rather than guessing. Don't claim high confidence on a
blurry photo.`;

export type ClassifyResult = {
  label: 'vape' | 'thc_vape' | 'not_a_vape' | 'uncertain';
  confidence: number;
  rationale: string;
  version: string;
  costMicros: number;
};

export async function classifyImage(jpeg: Buffer): Promise<ClassifyResult> {
  const t0 = Date.now();
  const resp = await anthropic.messages.create({
    model: process.env.CLASSIFIER_MODEL!,
    max_tokens: 200,
    system: [{ type: 'text', text: SYSTEM, cache_control: { type: 'ephemeral' } }],
    messages: [{
      role: 'user',
      content: [
        { type: 'image',
          source: { type: 'base64', media_type: 'image/jpeg',
                    data: jpeg.toString('base64') } },
        { type: 'text', text: 'Classify this image.' },
      ],
    }],
  });

  const text = (resp.content[0] as any).text.trim();
  const parsed = JSON.parse(text);

  // Cost in micro-USD. Haiku 4.5 pricing (verify current rates):
  //   input  ~$1.00 / Mtok      cache hit ~$0.10 / Mtok
  //   output ~$5.00 / Mtok
  const inTok       = resp.usage.input_tokens;
  const cachedInTok = (resp.usage as any).cache_read_input_tokens ?? 0;
  const outTok      = resp.usage.output_tokens;
  const costMicros  = Math.ceil(
    ((inTok - cachedInTok) * 1.00 + cachedInTok * 0.10 + outTok * 5.00)
  );

  return {
    label: parsed.label,
    confidence: parsed.confidence,
    rationale: parsed.rationale,
    version: `${process.env.CLASSIFIER_MODEL}:1`,
    costMicros,
  };
}

export async function computePHash(jpeg: Buffer): Promise<string> {
  return new Promise((resolve, reject) => {
    imageHash({ data: jpeg, ext: 'image/jpeg' }, 16, true,
              (err: any, hash: string) => err ? reject(err) : resolve(hash));
  });
}
The prompt caching on the system block matters — without it you pay full input rate on every call. With it, only the image tokens + output cost real money. Expect ~$0.0008–$0.0012 per image on Haiku 4.5.

4. Worker entry point
server/classifier/worker.ts:

import { db } from '../db';
import { classifyImage, computePHash, ClassifyResult } from './classify';

export async function processCapture(captureId: number) {
  const cap = await db.dropCaptures.findUnique({ where: { id: captureId } });
  if (!cap || cap.role !== 'after' || cap.classifierRanAt) return;

  const jpeg = await blobRead(cap.blobKey);
  const phash = await computePHash(jpeg);
  await db.dropCaptures.update({ where: { id: captureId }, data: { phash } });

  let result: ClassifyResult | null = null;
  let usedCache = false;

  // 1) pHash cache lookup
  const cutoff = new Date(Date.now() - Number(process.env.CLASSIFIER_DEDUPE_HOURS ?? 24) * 3600 * 1000);
  const prior = await db.dropCaptures.findFirst({
    where: { phash, classifierRanAt: { gte: cutoff }, id: { not: captureId } },
    orderBy: { classifierRanAt: 'desc' },
  });
  if (prior?.classifierLabel) {
    result = {
      label: prior.classifierLabel as any,
      confidence: prior.classifierConfidence ?? 0.5,
      rationale: 'cached',
      version: prior.classifierVersion + ':cached',
      costMicros: 0,
    };
    usedCache = true;
  }

  // 2) Budget guard
  if (!result) {
    const provider = process.env.CLASSIFIER_PROVIDER ?? 'off';
    const budgetUsd = Number(process.env.CLASSIFIER_DAILY_BUDGET_USD ?? 5);
    const today = new Date().toISOString().slice(0, 10);
    const usage = await db.classifierCostLog.findUnique({ where: { day: today } });
    const usedUsd = (usage?.totalMicros ?? 0) / 1_000_000;

    if (provider === 'off' || usedUsd >= budgetUsd) {
      // Phase 0 behavior — auto-accept, flag for review
      result = {
        label: 'uncertain', confidence: 0.5,
        rationale: provider === 'off' ? 'classifier_disabled' : 'daily_budget_exceeded',
        version: 'pass_through:1', costMicros: 0,
      };
    } else {
      result = await classifyImage(jpeg);
    }
  }

  // 3) Persist classifier output
  await db.dropCaptures.update({
    where: { id: captureId },
    data: {
      classifierLabel: result.label,
      classifierConfidence: result.confidence,
      classifierRanAt: new Date(),
      classifierVersion: result.version,
      classifierCostMicros: result.costMicros,
    },
  });

  if (result.costMicros > 0 && !usedCache) {
    await db.classifierCostLog.upsert({
      where: { day: today },
      update: { totalMicros: { increment: result.costMicros }, callCount: { increment: 1 } },
      create: { day: today, totalMicros: result.costMicros, callCount: 1 },
    });
  }

  // 4) Compute verdict (rules from REPLIT_BACKEND_PROMPT.md §3.2)
  const drop = cap.eventId
    ? await db.drops.findFirst({ where: { eventId: cap.eventId } })
    : null;
  if (!drop) return;  // drop row may arrive later; trigger on insert

  const bin = await db.bins.findUnique({ where: { id: drop.binId } });
  let accepted = true, reason = '';
  if (result.confidence < 0.55) {
    accepted = true; reason = 'low_confidence';
  } else if (result.label === 'not_a_vape' && bin.rejectNonVapes) {
    accepted = false; reason = 'not_a_vape';
  } else if (result.label === 'thc_vape' && bin.rejectThcVapes) {
    accepted = false; reason = 'thc_vape_rejected';
  }

  const reviewNeeded =
    result.label === 'uncertain' ||
    result.confidence < 0.7 ||
    result.version.startsWith('pass_through');

  await db.drops.update({
    where: { id: drop.id },
    data: {
      verdictReady: true,
      verdictAccepted: accepted,
      verdictReason: reason,
      verdictDecidedAt: new Date(),
      verdictReviewNeeded: reviewNeeded,
      pointsAwarded: accepted ? drop.pointsAwarded : 0,
    },
  });
}
Queue trigger: kick processCapture(id) from the drop_captures insert hook, and re-kick any matching captures when a drops row is inserted with an event_id (covers the "image arrived before drop POST" race).
